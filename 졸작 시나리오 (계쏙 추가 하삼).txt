
프로젝트 생성
	프로젝트 생성 일자 
		템플릿 엔진은 하나
		이름 추가 멤버 (권한) 


프로젝트 관리

	팀장과 팀원 두가지 자격
		팀장은 일정 관리와 역할 할당이 가능함
		팀원은 내용을 확인하고 완료 표시만 보낼 수 있음
		만약 팀원이 내용 편집을 하고 싶으면 채팅으로 요청함

	document 로그 (오픈 document main / user)
		내가 한일 이나 기술적 설명이 필요함
		

프로젝트 기능

	1. 메인 (전체 분류를 다보여지고 메인 창을 하나 정하자)
		프로젝트와 팀원의 소개 (Git MD 창 처럼)
		목표와 개인 목표(todolist) 
		팀이 해야될 작업 수와 완료된 작업수


	2. 목표 (목표 UI 참고)
		목표내용
		목표 참여자 
		목표 데드라인 (start finsh)		



	3. 역할 분담 체제 - 권한자가 할당 함
		
		to - do - list
		대시보드 형태 (to do list)
			클릭 시 모달창으로 띄워주며 /목표 내용 / 참여자 이름 / start finsh 일 / 
		
		워크 플로
			to do list를 캘린더 그래프 형식으로 보여줌
		
	3. (된다면) 프로젝트 전체 캘린더 (도표 형식으로 무조건 해야함)
		역할 분담 시 데드라인 날짜를 정하면 도표에 자동으로 띄우게 = 가시화를 위해
		
	4. 메시지함이던 소통할 수 있는 채팅 (우측 화면에 동시에 올려야 됨)
			멘션 처리 가능하게 (팀원이 팀장에게 의견 제시나 할 일 수정)

	5. 주석 처리 할 수 있는 오픈 document
